 1. Investment Transaction Processing Pipeline (Primary — Python)                                                                                       
                                                                                                                                                         
  Overview:                                                                                                                                              
  Designed and built an event-driven data pipeline processing daily mutual fund transactions from Registrar and Transfer Agents (RTAs). The system       
  handles complete transaction lifecycle — from file ingestion through lot tracking to capital gains computation — processing high-volume feeds with     
  complex business rules specific to Indian mutual fund regulations, while maintaining an architecture adaptable to any jurisdiction's tax and compliance
   requirements.                                                                                                                                         
                  
  Data Ingestion & Normalization:
  - Developed multi-format parsers for CAMS WBR2 (DBF with cp1252 encoding) and KFintech MFSD201 (pipe-delimited text, DBF) transaction feeds
  - Built transaction normalizer mapping 120+ RTA-specific transaction codes into a unified taxonomy (PURCHASE, REDEMPTION, SWITCH, STP, DIVIDEND,       
  TRANSFER, PLEDGE, BONUS, DEMAT)
  - Implemented reversal/deletion handling with mode-based filtering (Normal, Modified, Reversal, Deletion) and rejection suffix pattern matching        
                  
  FIFO Lot Matching Engine:                                                                                                                              
  - Built in-memory FIFO lot tracking engine using bisect for O(log n) sorted insertion — creates purchase lots on credits, depletes lots on debits using
   First-In-First-Out matching                                                                                                                           
  - Generates matched lot records capturing buy/sell linkage for realized gain/loss calculation
  - Handles corporate actions: switch-in/switch-out pairing, reversal matching across batches, transfer-in/out processing
                                                                                                                                                         
  Capital Gains Computation Engine:
  - Implemented comprehensive capital gains engine supporting 6 tax regimes (TR1–TR6) spanning regulatory changes from 2004 to 2024                      
  - Built grandfathering calculator for equity funds acquired before cutoff date — applies formula MIN(Sell, MAX(Buy, FMV)) using historical NAV as Fair
  Market Value                                                                                                                                          
  - Implemented cost inflation indexation using Cost Inflation Index (CII) tables for debt fund LTCG
  - Added stamp duty calculator, transitional relief, and holding period classification (STCG vs LTCG based on fund category and holding duration)
  - Fund category classification: Equity-oriented (>65%), Debt-oriented (<65%), Specified MF (<35%), Hybrid, Gold/FOF                                    
   
  Data Integrity & Performance:                                                                                                                          
  - Designed cross-batch deduplication using content hashing (dedup_hash) to ensure idempotent processing across overlapping feeds
  - Built switch/reversal matching algorithms using batch aggregation to pair related transactions
  - Optimized MongoDB writes using batch operations ($in queries, bulk_write) and $setWindowFields for server-side running balance calculation — reduced
  query volume by 95%
  - Implemented parallel enrichment tasks for NAV history, scheme master, benchmarks, and demographics

  Domain Expertise:
  Deep understanding of Indian mutual fund operations — RTA transaction lifecycles, SEBI regulations, tax regimes (grandfathering, indexation, LTCG/STCG
  classification), and fund categorization. Architecture designed with jurisdiction-agnostic tax rules engine (JSON-configurable) — adaptable to UCITS
  (EU), SEC (US), or other regulatory frameworks.

  Tech: Python, Apache Airflow (TaskFlow API), Polars, MongoDB (aggregation pipelines, $setWindowFields), Kafka, AWS S3, Decimal precision arithmetic
