1. Multiple/Single Dag: Since same worker pool executes both the dogs, single dag is preferable:
    1. Retry is easy
    2. Easy dependency management 
    3. One pipeline
    4. Scheduler performance will be better

2. DBF vs Mongo/Hashmap:
    1. It is recommended to normalise the fields from karvy and cms to single field names, to make parallel processing much faster.
    2. Mongo will be used as a single source of truth.
    3. It is okay to use hashmap as a temporary, per-task in-memory accelerator.

Why in memory Hashmap without discarding is bad:
	1. Airflow is built for:
        * stateless tasks
        * retrying from disk / DB
        * horizontal scaling
	2. A huge in-memory hashmap:
        * dies with the task
        * must be rebuilt on retry
        * cannot be shared across workers
        * cannot be checkpointed
    3. Mongo gives: 
        * cannot be checkpointed
        * persistence
        * resumability
        * retry safety
        * observability


3. Polars vs Pandas: Polars are much better than pandas:
Aspect	Polars	Pandas
Speed	5-10x faster	Baseline
Memory	More efficient	Higher usage
DBF Support	Via pandas or dbfread	Native via simpledbf
Learning Curve	Similar to Pandas	Well-known
Parallel Processing	Built-in	Requires Dask
